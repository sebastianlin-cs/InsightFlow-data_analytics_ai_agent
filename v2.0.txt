一、项目定位

InsightFlow 是一个全栈数据分析 AI Agent 平台。

用户可以：
1. 注册 / 登录
2. 上传 CSV / Excel 数据集
3. 系统把原始数据文件持久化保存到 backend/storage/
4. 系统生成 Data Catalog / Schema Context 并存入 PostgreSQL
5. 用户基于某个 dataset 创建 analysis session
6. 用户在 Agent Chat 中用自然语言向数据集提问
7. Agent 基于 dataset metadata、schema catalog、session messages、LLM planner、pandas tools 或 controlled code execution 生成分析结果

项目核心目标：
不是普通 CRUD，而是一个工程化的数据分析 Agent 平台，重点展示：
- 用户系统
- 数据资产管理
- Data Catalog
- Analysis Session
- LangGraph workflow
- LLM planning
- deterministic pandas tools
- chart generation
- agent trace / observability
- controlled code execution
- session persistence

核心架构原则：
PostgreSQL = 控制平面 / 上下文层
Local storage = 数据平面
dataset_id = 连接用户、原始数据、Data Catalog、session、messages、Agent workflow 的核心主键

当前数据存储策略：
- 原始 CSV / Excel 文件保存在本地：
  backend/storage/datasets/user_{user_id}/...
- PostgreSQL 只保存：
  users
  datasets metadata
  dataset_schemas / Data Catalog
  analysis_sessions
  analysis_messages
  structured_result_json / metadata
- 不把用户上传数据逐行导入 PostgreSQL
- 不动态创建 SQL table
- 所有 dataset/session/message 都必须通过 current_user 做 ownership 隔离

十一、v2.0 Code Contract

LLM 不能生成完整程序，只能生成：

def analyze(df):
    ...
    return {
        "answer": "...",
        "tables": {...},
        "charts": [],
        "metadata": {...}
    }

规则：
- df 已由系统加载
- LLM 不能读取文件
- LLM 不能连接数据库
- LLM 不能联网
- LLM 不能执行系统命令
- LLM 不能访问环境变量
- LLM 不能安装包
- LLM 不能访问任意路径
- LLM 不能写数据库
- LLM 不能多文件 join
- 只允许当前 session 的当前 dataset

允许库：
- pandas as pd
- numpy as np
- matplotlib.pyplot as plt

禁止模块：
- os
- sys
- subprocess
- socket
- requests
- pathlib
- shutil
- pickle
- multiprocessing
- threading

禁止函数：
- open
- eval
- exec
- compile
- input
- __import__
- globals
- locals
- vars
- dir
- getattr
- setattr
- delattr

禁止属性：
- __dict__
- __class__
- __bases__
- __subclasses__
- __globals__
- __code__

十二、v2.0 Code Generation Prompt 输出结构

LLM code generation 应输出 JSON：

{
  "code": "def analyze(df): ...",
  "explanation": "...",
  "columns_used": ["revenue", "region", "units"],
  "expected_result_keys": ["answer", "tables", "charts", "metadata"]
}

不要只输出 markdown code block。

十三、v2.0 Safety Validation

使用 AST 做静态安全检查。

输入：
{
  "code": "...",
  "allowed_function_name": "analyze",
  "allowed_imports": ["pandas", "numpy", "matplotlib", "matplotlib.pyplot"],
  "forbidden_imports": [...],
  "forbidden_calls": [...],
  "forbidden_attributes": [...]
}

输出成功：
{
  "passed": true,
  "reason": null,
  "blocked_items": [],
  "warnings": []
}

输出失败：
{
  "passed": false,
  "reason": "Forbidden import: os",
  "blocked_items": ["os"],
  "warnings": []
}

要求：
- code 必须 ast.parse 成功
- 必须只定义一个顶层函数 analyze
- analyze 必须只接收 df 一个参数
- 禁止危险 import/call/attribute
- 可检测时拦截 while True
- safety failed 时不得执行
- safety failed 不进行 reentry retry

十四、v2.0 Subprocess Runner

v2.0 第一版采用 subprocess runner，而不是 Docker sandbox。

Runner 责任：
- 不在 FastAPI 主进程直接 exec generated code
- 创建临时 workspace
- 写入 generated code script
- 子进程加载 dataset 为 df
- 子进程调用 analyze(df)
- 子进程写 result.json
- 主进程读取 result.json
- 捕获 stdout / stderr
- enforce timeout
- timeout 时 kill process
- 返回结构化 CodeExecutionResult

注意：
README 需要声明：
Current subprocess runner is a local controlled execution MVP, not a production-grade security sandbox.

Runner 输出：
{
  "status": "success" | "error" | "timeout" | "blocked",
  "answer": string | null,
  "tables": object,
  "charts": list,
  "metadata": object,
  "stdout": string,
  "stderr": string,
  "error": string | null,
  "execution_time_ms": int
}

十五、v2.0 Reentry / Debug

MVP 只做最多一次 retry：
max_debug_retry = 1

可以 retry：
- KeyError
- TypeError
- ValueError
- AttributeError
- NameError
- JSON serialization error
- matplotlib shape mismatch
- division by zero

不 retry：
- safety validation failure
- forbidden import/function
- timeout
- network/system access attempt
- repeated same error
- max retry reached

Reentry workflow:
first generated code
  -> AST safety validation
  -> execute
  -> if retryable runtime error:
       call LLM debug prompt once
       repaired code
       AST safety validation again
       execute repaired code once
  -> if fail again:
       stop and return structured error

Debug LLM 输出 JSON：
{
  "code": "def analyze(df): ...",
  "repair_explanation": "...",
  "changed_columns": {
    "removed": [],
    "added": []
  },
  "columns_used": [],
  "expected_result_keys": ["answer", "tables", "charts", "metadata"]
}

重要：
修复代码必须再次过 AST safety validation。
不得因为是 repair 就跳过安全检查。
