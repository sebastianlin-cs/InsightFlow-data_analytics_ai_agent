一、项目定位

InsightFlow 是一个数据分析 AI Agent 平台。

用户可以：
1. 注册 / 登录
2. 上传 CSV / Excel 数据集
3. 系统将数据集作为云端可复用的数据资产持久化保存
4. 系统生成 Data Catalog / Schema Context
5. 用户基于某个 dataset 创建 analysis session
6. 用户通过 Agent Chat 向数据集提问
7. Agent 基于 dataset metadata、catalog、session messages 进行分析和回答

项目核心目标不是普通 CRUD，而是一个具备工程化结构的数据分析 AI Agent 平台。

当前 demo scope：
1. 原始 CSV / Excel 文件存储在云服务器本地磁盘 / backend/storage/
2. PostgreSQL 保存用户、dataset metadata、Data Catalog、session、messages、Agent response context
3. 暂时不接 OSS / S3 / COS
4. 暂时不把用户上传的数据逐行导入 PostgreSQL
5. 暂时不动态创建 SQL table
6. 后续 Agent 分析时根据 dataset_id 找到 storage_uri 和 Data Catalog

核心架构原则：
PostgreSQL 是控制平面和上下文层；
本地文件存储是数据平面；
dataset_id 是连接用户、原始数据、Data Catalog、session、messages、Agent workflow 的核心主键。

v0.8 目标：
验证 Agent 请求可以：
1. 接收 query
2. 根据 session_id 找到 session
3. 根据 session.dataset_id 找到 dataset
4. 加载 dataset metadata
5. 加载 dataset_schemas / Data Catalog
6. 加载 recent messages
7. 生成 rule-based basic response
8. 保存 user message
9. 保存 assistant message
10. 返回 structured response

新增或已有文件大致包括：
- backend/app/api/agent.py
- backend/app/schemas/agent.py
- backend/app/services/agent_service.py
- backend/app/agent/state.py

POST /api/agent/query request：
{
  "session_id": 1,
  "query": "What columns are in this dataset?"
}

Response：
{
  "session_id": 1,
  "dataset_id": 12,
  "answer": "This dataset has 8 columns. Measures include revenue and profit...",
  "metadata": {
    "mode": "minimal_agent",
    "used_schema": true,
    "used_recent_messages": true,
    "message_saved": true
  }
}

v0.8 当前 Agent 能力：
- 如果 query 问 columns / schema / fields / catalog：
  返回 Data Catalog summary
- 如果 query 问 what can I analyze / summarize dataset：
  返回基于 row_count / column_count / measures / dimensions / time columns 的能力总结
- 否则：
  返回 fallback，说明已加载 dataset context 和 session history，真正分析工具会在后续版本加入

六、当前架构原则

后续继续遵守：

1. FastAPI 负责外部 API、鉴权、权限校验、调用 service / agent workflow
2. PostgreSQL 负责长期持久化 users / datasets / catalog / sessions / messages / traces / reports
3. Local storage 负责保存原始 CSV / Excel 文件
4. LangGraph 负责未来 Agent workflow
5. 不要把分析逻辑堆进 FastAPI route
6. 不要为每种分析任务暴露一堆外部 API
7. 核心 Agent API 应该保持 POST /api/agent/query

不要设计这些外部分析 API：
- /api/analyze/trend
- /api/analyze/groupby
- /api/chart/bar

这些未来应该是内部 tools / LangGraph nodes，而不是外部 API。
