Project Scope & In-Scope Task List
Project: PostgreSQL-backed Multi-source Financial / Enterprise Data Analysis Agent Platform

============================================================
1. Project Positioning
============================================================

Build a PostgreSQL-backed multi-source financial / enterprise data analysis agent platform.

Users can register, log in, upload Excel / CSV datasets, persist datasets in their personal workspace, view automatically generated Data Catalogs, and later select saved datasets for AI-assisted analysis.

The Agent uses a LangGraph-based workflow to load dataset context, classify intent, perform safety checks, plan tasks, call analysis tools, generate charts, produce final insights, and save all chat history, structured analysis results, tool traces, and reports into PostgreSQL.

Redis is intentionally out of the demo scope. Short-term caching, rate limiting, and async task coordination can be listed as future work.

Core engineering focus:
- User workspace
- Persistent dataset management
- Data Catalog generation
- Agent workflow orchestration
- Tool Harness
- Python-based data analysis
- Chart generation
- Safety guardrails
- PostgreSQL-backed chat history and trace persistence
- Resume-ready engineering demo

============================================================
2. User Experience Journey
============================================================

User opens platform
  -> Register / Log in
  -> Enter personal Workspace
  -> Upload Excel / CSV dataset
  -> System validates and stores file
  -> System parses dataset and generates Data Catalog
  -> User views dataset details and schema
  -> User selects dataset for analysis
  -> User asks natural language business / financial question
  -> Agent loads dataset schema and session context
  -> Agent classifies intent and checks safety
  -> Agent plans analysis tasks
  -> Agent calls tools through Tool Harness
  -> System displays table, chart, final insight, and agent trace
  -> User asks follow-up questions
  -> System loads previous session context from PostgreSQL
  -> User generates and saves report
  -> User later returns to reuse saved datasets and historical analysis sessions

============================================================
5. Final Scope Statement
============================================================

Build a PostgreSQL-backed multi-source financial / enterprise data analysis agent platform.

Users can register, log in, upload and save Excel / CSV datasets, and the system automatically parses uploaded files, persists dataset metadata, generates a Data Catalog, and allows users to reuse saved datasets in future AI analysis sessions.

The Agent uses a LangGraph workflow to load user-specific dataset context, classify user intent, perform safety checks, plan analysis tasks, call tools through a Tool Harness, run Python-based analysis, generate chart specifications, and produce grounded business insights.

All datasets, schemas, analysis sessions, chat messages, structured results, tool traces, and saved reports are persisted in PostgreSQL. The first demo scope intentionally excludes Redis and large-scale distributed infrastructure, while leaving caching, rate limiting, async processing, real external database connectors, and MCP integration as future work.
